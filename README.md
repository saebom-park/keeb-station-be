# ⌨️ keeb-station – Keyboard E-commerce Service

키보드 취미를 기반으로 한 **e-commerce 서비스**로,  
상품 옵션·재고·주문 흐름을 중심으로 실서비스 구조를 학습하고 구현하는 메인 프로젝트입니다.

Spring Boot + JPA를 기반으로 도메인 설계, 트랜잭션 처리, 인증/인가까지  
백엔드 전반을 깊이 있게 다루는 것을 목표로 합니다.

---

## 💡 시나리오

keeb-station은 **기계식 키보드 및 커스텀 파츠를 판매하는 쇼핑몰 서비스**입니다.

- 관리자는 키보드 상품과 옵션(스위치 타입, 키압, 배열 등)을 등록하고 재고를 관리합니다.
- 사용자는 상품 옵션을 선택하여 주문을 생성할 수 있습니다.
- 주문은 재고 차감 → 결제(모의) → 상태 변경 흐름을 가집니다.
- 인증은 JWT 기반으로 처리되며, 관리자와 일반 사용자 권한을 분리합니다.

본 프로젝트는 실제 키보드 쇼핑몰을 가정하여  
**옵션 기반 상품 구조와 주문·재고 흐름의 안정적인 처리**에 초점을 둡니다.

---

## 📋 요구사항

### 1. 회원 (Member)
- 이메일 기반 회원가입 및 로그인
- 비밀번호 암호화 저장
- JWT 기반 인증 (Access Token)
- ROLE_USER / ROLE_ADMIN 권한 분리

### 2. 상품 (Product / Option / Stock)
- 키보드 상품 등록 및 조회
- 상품별 옵션 구성 (예: 스위치 타입, 키압, 배열)
- 옵션 단위 재고 관리
- 재고 부족 시 주문 불가 처리

### 3. 주문 (Order)
- 주문 생성 (옵션 선택 가능)
- 주문 상태 관리 (CREATED, PAID, CANCELED)
- 주문 생성 시 재고 차감
- 주문 취소 시 재고 복원
- 트랜잭션 기반 처리

### 4. 보안
- Spring Security 기반 인증·인가
- JWT 토큰 검증 필터 적용
- 관리자 전용 API 접근 제한

---

## 🎯 구현 포인트

- 도메인 중심 패키지 구조 설계
- JPA 연관관계 매핑 및 트랜잭션 경계 명확화
- 옵션 단위 재고 차감 로직의 일관성 유지
- 인증/인가 로직과 비즈니스 로직 분리
- 기능 구현보다 **구조적 이해와 설계 의도**를 우선
